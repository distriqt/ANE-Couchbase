This class represents the Couchbase extension.

General

You should always check the isSupported flag of the extension before making any calls to the extension. This will return true on supported platforms and false otherwise. Ensuring your application correctly uses this flag means the application will port to all platforms. 

     try
     {
         if (Couchbase.isSupported)
         {
             // Do Couchbase operations
         }
         else
         {
             // Provide alternative fallback or inform user
         }
     }
     catch (e:Error)
     {
     }
Setup

The first step for using this extension is to initialise and setup the couchbase manager. The Couchbase manager must be setup by calling Couchbase.service.setup(). The example below shows the suggested method of this process. 

     try
     {
         trace( "Couchbase Supported: " + Couchbase.isSupported );
         trace( "Couchbase Version:   " + Couchbase.service.version );
              if (Couchbase.isSupported)
         {
         Couchbase.service.setup();
         }
     }
     catch (e:Error)
     {
     }
Listing available Databases

The Couchbase manager provides a function to retrieve all the available database names. The following example shows how to retrieve and trace out all the database names currently created. 

     var databases:Array = Couchbase.service.getAllDatabaseNames();
     for (var i:int = 0; i < databases.length; ++i)
     {
         trace( "db: " + databases[i] );
     }
Create or Retrieve a Database

The process to create or retrieve a database uses the same function call, i.e. getDatabase. If no matching database is found a new one is created and initialised. 

     var db:Database = Couchbase.service.getDatabase( "dbName" );
CRUD Operations

CRUD operations (create, read, update, delete) are provided on the Database and Document classes. See those classes for information on the functionality.

The following is a simple example showing the creation of a document and then adding some properties to the document. It assumes a database reference has already been retrieved. 

     var doc:Document = db.createDocument();
     
     doc.properties.someTestValue = 12345678;
     doc.properties.someTestString = "a test string";
     
     doc.commit();

getDatabase()method 
public function getDatabase(databaseName:String, options:DatabaseOptions = null):Database

This returns the Database with the given name. If the Database does not already exist, it is created.

  • The database name must begin with a lowercase letter.
  • The database name must contain only valid characters. The following characters are valid in database names:
    • Lowercase letters: a-z
    • Numbers: 0-9
    • Special characters: _$()+-/
The file for the database has a .cblite extension.

Parameters

databaseName:String — The name of the database to retrieve/create
 
options:DatabaseOptions (default = null)

Returns
Database — A valid Database object if successful or null if there was an error

Examples

The following example demonstrates the complete process of setup, retrieving a database, creating a document and adding a property to the document. 

     try
     {
         trace( "Couchbase Supported: " + Couchbase.isSupported );
         trace( "Couchbase Version:   " + Couchbase.service.version );
     
         if (Couchbase.isSupported)
         {
         Couchbase.service.setup();
             var db:Database = Couchbase.service.getDatabase( "myAppDB" );
             
             var doc:Document = db.createDocument();
             doc.properties.someValue = 12345678;
     
             doc.commit();
         }
     }
     catch (e:Error)
     {
     }
